PROJECT CONTEXT FOR AI MODELS & DEVELOPER REFERENCE

Project Name:
- Anas Ali Personal Portfolio & Creative Developer Website

Project Type:
- Pure Frontend Static Website (HTML5, Vanilla CSS3, Vanilla JavaScript, Web Components)
- No build step, no bundler, no heavy framework required
- Runs directly by opening index.html in any modern browser or hosting via GitHub Pages

Main Goal:
- Showcase personal brand, software development portfolio (Flutter, C++, Full-Stack), skills, freelance services, and contact links through a high-performance, visually stunning cyber-style UI with rich micro-animations.

==================================================
1) HOW THE UI IS LOOKING RIGHT NOW (VISUAL OVERVIEW)
==================================================

Theme & Color Palette:
- Dark Cyber / Sci-Fi Aesthetic: Sleek dark mode using deep slate and blue-black tones as background layers (--bg-color: #12181f, --bg-secondary: #1b232d).
- Glowing Neon Accents: Vibrant cyan and teal accents (--primary-accent: #3bc5b5, --secondary-accent: #2e9d92) create eye-catching focal points, borders, and hover glows.
- Typography: 
  - 'Orbitron' (Google Fonts) used for bold, futuristic headings, titles, and numbers.
  - 'JetBrains Mono' (Google Fonts) used for tech-inspired body copy, navigation, tags, and terminal typewriter effects.
- Visual Polish: Glassmorphism backdrops, subtle box-shadow glows, layered z-index compositions, and smooth drop-shadows on device mockups.

Page & Layout Structure:
1. Header / Top Contact Pill: A fixed glowing top-right "CONNECT" button linking to connect.html.
2. Hero Section (#hero):
   - Background: Animated SVG wave path ("river") with continuous scrolling uppercase text.
   - Left Side (Desktop) / Top (Mobile): Interactive profile avatar bubble (assets/Anas.png) that dynamically morphs its border-radius and shifts toward the mouse pointer on hover. Includes an animated status bubble ("ANAS ALI" + rotating taglines like "Building scalable Flutter apps").
   - Right Side: Glowing greeting ("Assalamu Alaikum, I am Anas") and a terminal-style typewriter prompt cycling through "developer" and "designer".
3. Bento Grid Section (#portfolio):
   - A modern 4-column CSS grid featuring glowing glassmorphism cards with staggered fade-in animations:
     - Card 1: About Me (Anas Ali) featuring professional title, bio paragraph, and core skill tags.
     - Card 2: Live Freelancer & GitHub Stats Widget powered by a custom Web Component (<global-freelancer-live-stats-widget>).
     - Card 3: Freelance Services list with interactive hover arrows and a "Get In Touch" CTA button.
     - Card 4: Interactive dual auto-scrolling Tech Stack Marquee (normal and reverse tracks) with glowing technology pills and SVG icons.
4. Featured Projects Section (#featured-projects):
   - Full-Width Horizontal Layout Hierarchy:
     - All project cards (Dev Pulse, Anas Space, We Two) permanently span the full horizontal grid width (`grid-column: 1 / -1`, class `.project-card-hero` or `.project-card-full`) in a horizontal split-pane design (alternating content/media left and right).
   - All cards feature interactive Vanilla JS Image Carousels (`.carousel-container`) with aspect-ratio-aware containers (`.carousel-landscape` 16:9, `.carousel-portrait` 9:16), semantic SVG navigation chevrons, and empty pagination dot indicators (`.dot`). Anas Space displays a single image without carousel navigation controls. We Two (mobile app) wraps its portrait carousel inside a `.mobile-mockup-frame` container to render as a scaled-down rounded mobile device mockup with a bezel and camera notch. Raw screenshots are styled with a 12px border-radius, subtle cyan border, and soft box-shadow glow.
   - On mobile screens (≤768px), cards smoothly stack vertically with the image/mockup positioned cleanly on top and content below.
5. Contact Section (#contact):
   - Clean call-to-action ("Let's Create Something") with outlined neon buttons for Email, LinkedIn, GitHub, and Fiverr.
6. Floating Bottom Navigation Dock (.nav-dock):
   - Fixed pill-shaped glassmorphism bar at the bottom center with links to Home, Portfolio, Projects, Contact, and a "Download CV" button.
   - Automatically highlights the active section during scrolling via IntersectionObserver.
7. Standalone Connect Page (connect.html):
   - Dedicated social hub page featuring a header, back link, mini morphing bubble, and a responsive 12-column Bento grid with compact cards for LinkedIn, GitHub, Instagram, Email, Fiverr, and WhatsApp.

4) DEEP DIVE: HOW TO EDIT THE BENTO GRID (#portfolio)
==================================================

The Bento Grid is defined in index.html (lines 57-164) and styled in styles.css (lines 397-784).
It uses CSS Grid: `grid-template-columns: repeat(4, 1fr);` on desktop.

How Cards are Positioned:
- Card 1 (.bento-item-1): Spans 2 columns (`grid-column: span 2;`).
- Card 2 (.bento-item-2): Spans 2 columns and 2 rows (`grid-column: span 2; grid-row: span 2;`).
- Card 3 (.bento-item-3): Spans 1 column (`grid-column: span 1;`).
- Card 4 (.bento-item-4): Spans 1 column and 2 rows (`grid-column: span 1; grid-row: span 2;`).

How to Add or Modify a Bento Card:
1. To change Card 1 text or tags: Edit index.html lines 60-67. Add new tags inside `<div class="bento-tags">` using `<span class="tag">TagName</span>`.
2. To change Card 2 (Live Stats): The `<global-freelancer-live-stats-widget handle="AnasAli-2722">` tag automatically fetches stats. To modify its appearance or behavior, edit global-freelancer-live-stats-widget.js.
3. To change Card 3 (Services): Edit the unordered list in index.html lines 80-85 (`<ul class="services-list">`). Each `<li>` automatically receives a cyan triangle bullet (▶) via CSS `::before`.
4. To add technologies to Card 4 (Marquee):
   - Add a new pill inside `.tech-track` in index.html (lines 95-137 or 141-159):
     `<div class="tech-pill tech-yourtech"><span class="tech-pill-icon"><svg>...</svg></span><span class="tech-pill-label">YourTech</span></div>`
   - In styles.css, add an icon color rule around line 756: `.tech-yourtech .tech-pill-icon { color: #hexcolor; }`
   - Note: Duplicate the pills within the track to ensure a seamless infinite loop animation!

5) DEEP DIVE: HOW TO EDIT FEATURED PROJECTS (#featured-projects)
==================================================

The Featured Projects section uses a FULL-WIDTH HORIZONTAL LAYOUT HIERARCHY:
- Every project card (.project-card-hero, .project-card-full) permanently sits full width across the grid (`grid-column: 1 / -1`) with a horizontal split-pane design.
- To create a dynamic visual rhythm, the cards alternate their layout order:
  - Dev Pulse: Content Left | Media Right
  - Anas Space: Media Left | Content Right
  - We Two: Content Left | Media Right

Card Architecture:
Every project card contains a `.project-split` flex container with two children:
- `.project-split-content`: Title, description, tech tags, and a `<a class="project-details-btn">` button.
- `.project-split-media`: Image placeholder with gradient background holding an interactive `.carousel-container` (or wrapped in `.mobile-mockup-frame` for mobile apps like We Two).

How to Add a New Full-Width Project Card with Carousel:
1. In index.html inside `<div class="projects-grid">`, add:
   ```html
   <article class="project-card project-card-full reveal-on-scroll" data-reveal-delay="300">
       <div class="project-split">
           <div class="project-split-media">
               <div class="project-image-placeholder your-project-image">
                   <div class="carousel-container carousel-landscape" aria-roledescription="carousel">
                       <div class="carousel-track">
                           <div class="carousel-slide active"><img src="assets/Your_Project/ss1.png" class="carousel-slide-img"></div>
                           <div class="carousel-slide"><img src="assets/Your_Project/ss2.png" class="carousel-slide-img"></div>
                       </div>
                       <button class="carousel-btn carousel-btn-prev">...</button>
                       <button class="carousel-btn carousel-btn-next">...</button>
                       <div class="carousel-indicators">
                           <div class="dot active"></div>
                           <div class="dot"></div>
                       </div>
                   </div>
               </div>
           </div>
           <div class="project-split-content">
               <h3>Your Project Name</h3>
               <p>Short description of your project.</p>
               <div class="project-tech-pills">
                   <span class="project-tech-tag">Flutter</span>
                   <span class="project-tech-tag">Firebase</span>
               </div>
               <a href="projects/your-project.html" class="project-details-btn">
                   View Project Details
                   <span class="btn-glow" aria-hidden="true"></span>
               </a>
           </div>
       </div>
   </article>
   ```
2. In styles.css, define a custom gradient background for your placeholder:
   ```css
   .your-project-image {
       background:
           linear-gradient(150deg, rgba(59, 197, 181, 0.2), transparent 56%),
           radial-gradient(circle at 80% 20%, rgba(46, 157, 146, 0.25), transparent 52%),
           linear-gradient(130deg, #17212b, #10171f);
   }
   ```
3. Create a dedicated detail page by duplicating `projects/anas-space.html`.
4. The expand/collapse JS in script.js automatically detects new `[data-expandable]` cards — no JS changes needed.

Interactive Expand/Collapse Behavior:
- Clicking anywhere on a standard card (except the button) toggles the `.expanded` class.
- Only one card can be expanded at a time (others auto-collapse).
- Expand is disabled on mobile (≤768px viewport) where cards already stack vertically.
- A debounced resize listener auto-collapses cards when the viewport shrinks past the mobile breakpoint.

6) IMPORTANT GUIDELINES FOR AI MODELS & DEVELOPERS
==================================================

1. Preserve Pure Frontend Architecture: Do not introduce Node.js build tools, npm bundlers, TailwindCSS, or React/Next.js unless explicitly instructed by the user. Keep all code vanilla and static.
2. Maintain Section IDs & Class Hooks:
   - Section IDs (`#hero`, `#portfolio`, `#featured-projects`, `#contact`) MUST be preserved as they are actively monitored by the IntersectionObserver in script.js to highlight the floating navigation dock.
   - Preserve JS class hooks: `.bento-item`, `.reveal-on-scroll`, `.nav-dock-item`, `#typewriter`, `#bubble-tagline`, `.project-card[data-expandable]`, `.project-details-btn`, `.carousel-container`, `.carousel-btn`, `.carousel-dot`.
3. Design Aesthetics & Consistency:
   - When adding new UI components, always utilize existing CSS variables from `:root` (`--bg-secondary`, `--primary-accent`, `--border-color`, `--glow-color`).
   - Ensure all interactive elements (cards, buttons, links, tags) feature smooth hover states, subtle transforms, and glowing box-shadows to maintain the rich cyber aesthetic.
4. Responsive Integrity:
   - Always verify layout behavior across desktop (>1024px), tablet (768px–1024px), and mobile (<768px).
   - Ensure multi-column grids (Bento grid and Projects grid) gracefully collapse into single columns on smaller screens without horizontal overflow.
5. Documentation Updates:
   - If new sections, standalone pages, or major UI components are added, update this file (`PROJECT_CONTEXT_FOR_AI.txt`) and `README.md` to keep documentation accurate.
